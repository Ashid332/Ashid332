## 🚀 Production-Grade Featured Projects

### 🤖 DocuMind — LLM Document Q&A
*A high-accuracy RAG platform that enables users to query dense PDFs and documentation while verifying sources interactively.*
* **Business Problem**: Manual retrieval of contract terms or engineering specs from massive documentation sets was error-prone, costing operations teams average search times of 15 minutes per query and suffering from model hallucinations.
* **Measurable Impact**: **Reduced query turnaround from 15 minutes to 10 seconds** and achieved **92% source citation accuracy** with zero hallucinations.
* **Architecture**: Implemented a recursive character text splitter to parse documents, mapped embeddings via `OpenAI Text-Embedding-3-Small`, indexed them in `FAISS` vector DB, and orchestrated responses using a `LangChain` conversational QA chain. Serves APIs via a high-performance `FastAPI` server.
* **Engineering & Scalability Challenges**: Mapped direct document source page coordinates during chunking. Solved context window limitations and cost overheads by applying parent-document retrieval to fetch only the exact relevant paragraphs rather than entire file contexts.
* **Links**: [GitHub Repository](https://github.com/Ashid332/documind) | [Architecture Diagram](./docs/documind-architecture.png)

---

### 📊 DataVault — Self-Service Analytics Platform
*A drag-and-drop MERN-based analytics suite designed for non-technical business partners to create and view statistical distributions instantly.*
* **Business Problem**: The marketing and product growth teams had to write SQL or wait up to 3 days for analytics engineers to build standard dashboards, creating a major lag in optimization feedback loops.
* **Measurable Impact**: **Reduced analytics engineering tickets by 60%** and accelerated custom report visualization from **3 days to under 60 seconds**.
* **Architecture**: Responsive `React` and `Tailwind CSS` frontend paired with a modular `Node.js` & `Express.js` backend server. Uploaded CSVs are parsed and stored inside `PostgreSQL`. Plotly engines generate interactive chart structures dynamically based on auto-identified schemas.
* **Engineering & Scalability Challenges**: Large CSVs (up to 100MB) caused memory allocation faults and blocked the Node.js single thread. Resolved this by introducing backend stream parsing, piping file chunks directly to the database, and wrapping parsing logic in a non-blocking asynchronous queue.
* **Links**: [GitHub Repository](https://github.com/Ashid332/datavault) | [Interactive Demo](https://datavault.ashid.dev)

---

### 📉 Customer Churn Predictor
*A machine learning model and production API predicting client turnover weeks ahead of contract renewal.*
* **Business Problem**: Customer churn was leaking recurring revenue, and account managers lacked early warnings or diagnostic details to proactively engage at-risk customers.
* **Measurable Impact**: Achieved **84% prediction accuracy** with an **AUC-ROC of 0.81**, giving account managers a 30-day window to deploy retention plans.
* **Architecture**: Built using `Python`, `Pandas`, `NumPy`, and `Scikit-learn`. Trained an optimized `XGBoost` classifier. Built a local `SHAP` layer to output mathematical features weighting why a specific profile is classified at-risk. Wrapped in `Docker` for deployment.
* **Engineering & Scalability Challenges**: The historical churn dataset suffered from high class imbalance (10% churned vs 90% active). Implemented SMOTE (Synthetic Minority Over-sampling Technique) to stabilize model training. Designed the API to return feature explanations in under 50ms.
* **Links**: [GitHub Repository](https://github.com/Ashid332/customer-churn-predictor) | [Model Card](./docs/churn-model-card.md)

---

### 📈 Sales Intelligence ETL & Dashboard
*An enterprise-grade reporting engine automating data collection from transaction schemas to interactive executive views.*
* **Business Problem**: Sales teams and management were manually downloading, cleaning, and copy-pasting disconnected spreadsheets weekly, delaying critical inventory ordering.
* **Measurable Impact**: Saved management **6 hours per week in manual labor** and reduced executive sales reporting latency by **40%**.
* **Architecture**: Built python-based ETL scripts scheduled on `GitHub Actions` that query multiple transaction databases, normalize schemas using Pandas, clean duplicates, and load records into a centralized `PostgreSQL` instance. Integrated direct live query feeds to `Power BI`.
* **Engineering & Scalability Challenges**: Addressed slow dashboard load times caused by loading millions of row scans. Optimized database indexes and completely rewrote the dashboard logic using optimized Power BI DAX measures, bringing query execution under 2 seconds.
* **Links**: [GitHub Repository](https://github.com/Ashid332/sales-intelligence-dashboard)
